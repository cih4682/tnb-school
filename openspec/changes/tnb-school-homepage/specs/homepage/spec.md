## ADDED Requirements

### Requirement: Hero 섹션은 배경 영상과 핵심 카피, CTA를 제공한다
홈페이지 최상단의 Hero 섹션은 `t&b.mp4`를 풀스크린 배경(자동재생·음소거·반복)으로 표시하고, 서비스 핵심 카피와 "앱 둘러보기" CTA 버튼을 제공해야 한다(SHALL). 영상 위에는 가독성을 위한 어두운 오버레이가 적용되어야 한다(MUST).

#### Scenario: 페이지 진입 시 영상 자동 재생
- **WHEN** 사용자가 홈페이지에 처음 접속한다
- **THEN** Hero 영상이 음소거 상태로 자동 재생되며, 카피와 CTA 버튼이 영상 위에 표시된다

#### Scenario: CTA 클릭 시 앱 갤러리로 스크롤
- **WHEN** 사용자가 "앱 둘러보기" 버튼을 클릭한다
- **THEN** 페이지가 앱 갤러리 섹션으로 부드럽게 스크롤된다

### Requirement: 가치 제안 3카드 섹션을 표시한다
시스템은 Hero 바로 아래에 서비스의 핵심 가치 3가지를 카드 형태로 표시해야 한다(SHALL).

#### Scenario: 가치 카드 렌더링
- **WHEN** 사용자가 가치 제안 섹션에 도달한다
- **THEN** 3개의 카드(아이콘 + 제목 + 한 줄 설명)가 가로로 정렬되어 보인다

### Requirement: 앱 카운터는 현재/목표 개수를 시각화한다
시스템은 현재 등록된 앱 수와 목표(100)를 진행 바와 숫자로 보여야 한다(SHALL). 현재 앱 수는 `data/apps.ts`의 항목 개수에서 자동 계산되어야 한다(MUST).

#### Scenario: 카운터 표시
- **WHEN** 사용자가 카운터 섹션에 도달한다
- **THEN** "현재 N / 목표 100" 텍스트와 N/100 비율의 진행 바가 표시된다

### Requirement: 앱 갤러리는 카테고리 필터와 카드 그리드를 제공한다
시스템은 앱 목록을 카드 그리드로 표시하고, 카테고리(수업준비·평가·학생관리·자료제작) 필터를 제공해야 한다(SHALL). 각 카드는 제목, 카테고리, 짧은 설명, 아이콘 또는 썸네일을 포함해야 한다(MUST).

#### Scenario: 카테고리 필터링
- **WHEN** 사용자가 특정 카테고리 필터를 클릭한다
- **THEN** 해당 카테고리에 속한 앱 카드만 그리드에 표시된다

#### Scenario: 전체 보기
- **WHEN** 사용자가 "전체" 필터를 클릭한다
- **THEN** 모든 앱 카드가 표시된다

### Requirement: 티처버프 입학 안내 섹션을 제공한다
시스템은 입학금, 포함되는 혜택, 가입 CTA를 가격 카드 형태로 표시해야 한다(SHALL).

#### Scenario: 가격 카드 표시
- **WHEN** 사용자가 입학 안내 섹션에 도달한다
- **THEN** 입학금 금액, 혜택 리스트, "입학하기" CTA 버튼이 표시된다

### Requirement: 커스텀 앱 신청 폼을 제공한다
시스템은 선생님이 원하는 커스텀 앱을 신청할 수 있도록 이름·이메일·요청 내용 입력 폼을 제공해야 한다(SHALL). 제출 시 클라이언트 측에서 검증 후 확인 메시지를 표시해야 한다(MUST).

#### Scenario: 폼 제출
- **WHEN** 사용자가 모든 필수 필드를 채우고 "신청하기"를 클릭한다
- **THEN** 검증을 통과하면 "신청이 접수되었습니다" 확인 메시지가 표시된다

#### Scenario: 필수 필드 누락
- **WHEN** 사용자가 필수 필드를 비운 채 제출한다
- **THEN** 해당 필드 아래에 오류 메시지가 표시되고 제출되지 않는다

### Requirement: FAQ와 푸터 섹션을 제공한다
시스템은 자주 묻는 질문(아코디언 형식)과 푸터(저작권·연락처)를 페이지 하단에 표시해야 한다(SHALL).

#### Scenario: FAQ 항목 펼치기
- **WHEN** 사용자가 FAQ 질문을 클릭한다
- **THEN** 해당 질문의 답변이 아래에 펼쳐진다

### Requirement: 앱 데이터 모델
앱 데이터는 `data/apps.ts`에 정의되며, 각 앱은 `id`, `name`, `category`, `description`, `icon`(선택), `isNew`(선택) 필드를 가져야 한다(MUST). 카테고리는 `lesson-prep`, `assessment`, `student-management`, `material` 중 하나여야 한다(MUST).

#### Scenario: 앱 데이터 참조
- **WHEN** 갤러리 컴포넌트가 렌더링된다
- **THEN** `data/apps.ts`에서 앱 배열을 import하여 카드로 매핑한다
